Цикл while

Цикл while має наступний синтаксис: 

                
while (condition) {
    // код, який  також називаються "тілом циклу"
}

Код із тіла циклу виконується доти, доки умова істинна. Одне виконання тіла циклу називається ітерацією. 

                
let i = 0;
while (i < 3) { // виводить 0, потім 1, потім 2
    alert (i); 
    i++;
}

У прикладі вище, код буде виконуватись доти, доки змінна менша вказаного значення. Якби рядок "i++" був би відсутнім, то теоретично, цикл повторювався б вічно. На практиці ж браузер дасть користувачу можливість зупинити скрипт, що "завис".

Умовою циклу може бути будь-який вираз або змінна. Умова циклу обчислюється і перетворюється у логічне значення.

Можливі деякі коротші записи. Наприклад, замість умови while(i != 0), можна використовувати while(i). 

                
let i = 3;
while (i) { //коли i буде дорівнювати 0, буде false і цикл зупиниться.
    alert (i);
    i--;
}

Також, якщо тіло циклу складається лише з однієї інструкції, ми можемо забрати фігурні дужки: 

                
let i = 3;
while (i) alert (i--);

Цикл do...while

Використовуючи синтаксис do...while можна розмістити перевірку умови перед тілом циклу. 

                
do {
   //тіло циклу
 while (condition)

Цикл спочатку виконає тіло, а вже потім перевірить умову, і, поки її значення дорівнюватиме true, цикл буде виконуватись знову і знову. 

                
let i = 0;
do {
    alert (i);
    i++;
} while (i < 3);

Така форма синтаксису виправдана, якщо потрібно, щоб тіло циклу виконалось хоча б один раз, навіть якщо умова виявиться неістинною. На практиці частіше виконується форма з передумовою while(...) {}.

Цикл for

Цикл for складніший, але при цьому найбільш поширений. 

                
for (let i = 0; i < 3; i++) { //виведе 0, потім 1, потім 2
    alert (i);
}

Перший вираз у круглих дужках - це початок циклу, другий вираз - умова, а третій - крок. Умова перевіряється перед кожною ітерацією циклу, і, якщо результатом обчислення буде false, цикл зупиниться.

У розглядуваному прикладі змінна лічильника "i" була оголошена прямо в циклі. Це так зване "вбудоване" оголошення змінної. Такі змінні існують лише всередині циклу.

Також, замість оголошеної змінної можна використовувати вже існуючу і вона буде доступна після циклу, так як була оголошена зовні циклу. 

                
let i = 0;

for (i = 0; i < 3; i++) {
    alert (i); // 0, 1, 2
}

alert (i); // 3

Будь-яку частину в круглих дужках циклу for можна пропускати: початок, умову, крок. При цьому, крапки з комами обов'язково повинні бути, інакше буде синтаксична помилка. Також, якщо забрати всі частини, ми отримаємо нескінченний цикл. 

                
for (; ;) {
    //цикл буде виконуватись вічно
}

Директива break

Використовуючи спеціальну директиву break, ми можемо вийти з циклу в будь-який момент. 

                
let sum = 0;
while (true) {
    let value = +prompt ("Введіть число:", "");
    if (!value) break;
    sum += value;
}

alert ("Сума: " + sum);

У прикладі вище користувач щоразу вводить значення "і", якщо воно в умові даватиме false, то цикл припиняється завдяки директиві break і виконання переходить за межі циклу, а в даному прикладі на рядок alert.

Для закріплення можна навести ще один приклад: 

                
let num = 0;

for(; num < 5; num++) {
    alert (num);
    if (num == 2) break;
}

alert (`Робота завершена, num = ${num}`);

Використання "нескінченного" циклу разом із break є дуже зручним для ситуацій, коли умова, по якій треба перерватись, знаходиться не на початку чи кінці циклу, а посередині, чи в декількох місцях циклу.

Директива continue

Використання директиви continue, не перериваючи цикл, переходить до наступної його ітерації, якщо умова все ще дорівнює true. 

                
for (let i = 0; i < 10; i++) {
    if (i % 2 == 0) continue;
    alert (i); // спочатку 1, потім 3, 5, 7, 9
}

Цю директиву використовують, коли потрібно пропустити певну ітерацію циклу. Наприклад, у прикладі вище директива continue використовується, щоб виводити лише непарні числа. Для парних чисел директива continue припиняє виконання тіла циклу і передає керування на наступну ітерацію for (із наступним числом). Відповідно, таким чином, виводяться лише непарні значення.

Директива continue дозволяє уникати вкладеності. Наприклад, цикл, який виводить лише непарні значення, міг би виглядати так: 

                
for (let i = 0; i < 10; i++) {
    if (i % 2) {
        alert (i);
    }
}

Технічно, такий цикл ідентичний до попереднього циклу із директивою continue, але в даному прикладі ми отримали додатковий рівень вкладеності фігурних дужок і у випадку, якби код всередині if був би довший, це б погіршувало б читання, на відміну від варіанту з директивою continue.

Директиви break/continue не можна використовувати в умовному (тернарному) операторі ? в якості виразів, оскільки це викличе помилку. Це ще одна з причин не використовувати умовний (тернарний) оператор ? замість if.

Мітки для break/continue

Бувають випадки, коли потрібно вийти з декількох рівнів циклу за раз. Але, скажімо, у випадку, коли ми перебуваючи в циклі, що вкладений в інший цикл, використаємо break, ми перервемо лише внутрішній цикл і перейдемо у зовнішній. Для того, аби вийти за межі обидвох циклів можна використати мітки.

Мітка має вигляда ідентифікатора з двокрапкою перед циклом. 

                
outer : for (let i = 0; i < 3; i++) {
    for (j = 0; j < 3; j++) {
        let input = prompt (`Значення на координатах ${i}, ${j}:`, "");
        // якщо пусти рядок чи відміна, то вийти з обох циклів
        if (!input) break outer;
        // код циклу
    }
}
alert ("Готово!");

В даному вище прикладі ми бачимо, що викликом break outer ми переходимо до найближчого зовнішнього циклу із міткою outer:, розриваємо його і після цього керування переходить за межі циклу.

Мітку можна розміщувати на окремому рядку. 

                
labelName: for (let i = 0; i < 3; i++) {
    // код циклу
}

Директиву continue також можна використовувати із міткою. В такому випадку керування перейде а наступну ітерацію циклу з міткою.

Мітки не дають можливості передавати керування в довільне місце коду. Директива break повинна завжди знаходитись всередині блоку кода і за допомогою міток можна передавати керування до найближчого зовнішнього блоку із вказаною міткою. Технічно, для цього підходить будь-який маркований блок. 

                
label : {
    // код блоку
    break label; // зпрацює
    // код блоку
}

Хоча директиву break прийнято використовувати всередині циклів. В свою чергу директива continue працює лише всередині циклів.
