Типи даних

Значення змінних завжди відносяться до певного типу даних.

Оскільки JS є динамічно типізованою мовою, тип даних змінної змінюється динамічно в момент присвоєння або зміни значення, але аж ніяк не в момент оголошення.

Отже, типи даних в JS є, але змінні не прив'язані до них.

Використовуючи оператор typeof, можна отримати поточний тип даних змінної.

У JS існує 8 окремих типів даних:

  • undefined;
  • null;
  • boolean;
  • number;
  • bigInt;
  • string;
  • symbol;
  • object.

Також можна виділити в якості дев'ятого типу даних тип function - по суті є типом даних object, але виведений в окремий тип для простішого визначення типу для функції.

Тип даних undefined

Undefined - це невизначений тип.

Змінна поверне цей тип даних, якщо була оголошена, але їй не було присвоєно значення.

Відповідно, тип цей тип даних містить одне єдине значення - undefined.

Даний тип даних використовується для перевірки, чи була змінна визначена, чи ні.

Тип даних null

Даний тип даних, як і undefined містить лише одне значення - null.

У JS null не є посиланням на неіснуючий об'єкт, чи нульовим посиланням, як в деяких інших мовах програмування - null є спеціальним значенням, що позначає "нічого", "пусто".

Не варто плутати null та undefined: змінна із типом даних null - визначена, просто пуста, в той час як змінна із типом undefined - невизначена.

Якщо спробувати отримати тип даних змінної зі значенням null, ми отримаємо тип даних object - це офіційно признана помилка оператора typeof, яка тягнеться ще з часів створення JS і залишена для сумісності. Null не є об'єктом - це спеціальне значення із окремим типом.

Тип даних boolean

Булевий, або логічний тип даних.

Може приймати лише два значення: true або false. Булеві значення також можуть бути результатами порівнянь.

Тип даних number

Представляє значення у вигляді цілих чисел та у вигляді чисел із плавачою крапкою.

Окрім звичайних числових значень існують так звані спеціальні числові значення, які також відносять до типу даних number:

  • Infinity - представляє собою математичну нескінченність; відповідно, це значення більше будь-якого числа; це значення можна задати явно, або отримати в результаті ділення на 0;
  • -Infinity - те саме, лише зі знаком мінус і відповідно менше будь-якого числа;
  • NaN - представляє обчислювальну помилку; результат неправильної чи невизначеної математичної операції; будь-яка математична операція із NaN повертає NaN, відповідно, якщо десь у математичному виразі є NaN, то воно поширюється на весь результат, але виключення становить оператор піднесення до степеня: NaN**0=1.

Тип даних bigInt

Дає можливість працювати з цілими числами довільної довжини.

Щоб створити значення типу bigInt необхідно додати n в кінець числового літералу.

В даний момент bigItn підтримується лише у браузерах Firefox, Chrome, Edge, Safari, але не підтримується у IE.

У JS тип number не може безпечно працювати із числами більшими ніж (25^3 - 1) або меншими ніж -(25^3 - 1). За межами даного безпечного діапазону більшість чисел не можуть бути коректно представленими через обмеження, що пов'язані із внутрішнім представленням чисел у двійковій формі. Наприклад, непарні числа за межами безпечного діапазону будуть закруглятися до парних чисел. В той же час, деякі парні числа технічно зберігати можна, але робити цього не рекомендується.

Тип даних string

Визначає рядок тексту.

У JS рядок повинен братися в лапки.

У JS є три типи лапок:

  • одинарні;
  • подвійні;
  • зворотні.

Одинарні та подвійні лапки є простими і між ними немає різниці.

Зворотні лапки дозволяють вбудовувати вирази, змінні, значення у рядок, використовуючи ${...}.

Варто зазначити, що у JS не існує окремого типу даних для одного символа - є лише тип string. Рядок може мати ноль символів (тобто бути пустим), один символ або безліч.

Тип даних object

Відрізняється від усіх інших примітивних типів - їхніми значеннями можуть бути лише прості значення, а в об'єктах, в свою чергу, можна зберігати колекції даних, або більш складні структури.

Об'єкт може бути створений за допомогою фігурних дужок і може містити в собі набір необов'язкових властивостей, що розділяються між собою комами.

Властивість - це пара: "ключ: значення".

Тип даних symbol

Виконує роль унікального ідентифікатора об'єкта. Також є примітивним типом даних.

Нові символи створюються за допомогою однойменної функції.

Тип даних function

Як вже говорилося, цей тип даних є по суті типом object, який для зручності вивели в окремий тип для простого визначення функції.

Перетворення типів

У JS можна перетворювати один тип в інший за допомогою операторів, або спеціальних функцій.

Використовуючи функцію String можна перетворити змінну одного типу даних у рядкову.

Також можна перетворити рядок в число, використовуючи функцію Number. Якщо перетворювати в число рядок, у якому немає жодного числа, отримаємо NaN.

У JS можливе автоматичне перетворення без використання функцій. Наприклад, при діленні рядка "42" на рядок "2" отримаємо число 21.

Використовуючи функцію Boolean, можна здійснити перетворення змінної в булевий тип даних. При перетворенні будь-якого числа, окрім 0, чи рядка, окрім пустого, в булевий тип, отримаємо значення true, а при перетворенні числа 0 чи пустого рядка отримаємо false.
