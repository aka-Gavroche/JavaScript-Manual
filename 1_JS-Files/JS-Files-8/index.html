Функції призначені для виконання одного і того є коду у різних місцях.

У JS існують вбудовані функції, такі як alert(message), prompt(message, default), confirm(question), але можна створювати свої власні функції.

Оголошення функцій (Function declaration)

Оголошення функцій використовується для створення функцій. 

                
function showMessage () {
    alert ("Hello!");
}

Спочатку йде ключове слово function, потім ім'я функції, після чого йде список параметрів і вже саме тіло функції.

Створена функція може бути викликаною по її імені. Викликати функцію можна як після її оголошення, так і до цього, а також функцію можна викликати багато разів. 

                
function showMessage () {
    alert ("Hello!");
}

showMessage (); // цей виклик виконає код функції

Даний приклад яскраво демонструє одне з головних призначень функції: уникнути дублювання коду. Тепер, якщо потрібно буде змінити повідомлення, або спосіб його виводу, достатньо внести зміни лише в самій функції.

Імена функцій

Оскільки функція - це дія, то іменами функцій зазвичай є дієслова. Ім'я функції повинно бути коротким і описувати, що робить функція. В імені функції використовуються дієслівні префікси, що позначають дію, а після них йдуть вже самі об'єкти дій.

Наприклад деякі дієслівні префікси:

  • "show..." - функція щось показує;
  • "get..." - функція повертає якесь значення;
  • "calc..." - функція щось обчислює;
  • "create..." - функція щось створює;
  • "check..." - функція щось перевіряє і повертає логічне значення.

Приклади імен функцій: 

                
showMessage (); // показує повідомлення
getAge (); // повертає вік
calcSum (); //обчислює суму і повертає результат
createForm (); //створює форму і зазвичай повертає її
checkPermission (); //щось перевіряє і повертає логічне значення

Завдяки префіксам стає одразу зрозуміло, що саме робить функція.

Функція має виконувати лише те, що явно розуміється її назвою. Одна функція - одна дія.

Дві незалежні дії мають зазвичай дві функції - кожна дія має свою функцію, навіть якщо передбачається, що вони будуть викликатися разом (у цьому випадку можна створити третю функцію, яка буде їх викликати).

Декілька прикладів, що порушують правило:

  • getAge - буде поганим вибором, якщо функція буде виводити alert із віком, бо дана функція має лише повертати його, а для виведення отриманого значення ліпше використати іншу функцію (наприклад showAge);
  • createForm - буде поганим вибором, якщо функція буде змінювати документ, додаючи форму у нього, бо дана функція повинна лише створювати форму і повертати її;
  • checkPermission - буде поганим вибором, якщо функція буде відображати повідомлення з текстом "Доступ дозволено"/"Доступ заборонено", бо дана функція повинна виконувати лише перевірку і повертати результат, а для виведення тексту ліпше використати іншу функцію (наприклад showPermission).

Імена функцій, що використовуються дуже часто, роблять інколи дуже короткими. Наприклад фреймворк jQuery визначає функцію за допомогою символу "$". В бібліотеці Lodash основна функція представлена іменем "_".

Вкладеність і видимість функцій

Вкладена функція - це функція, що створюється в середині іншої функції. 

                
function getSumm () { //батьківська функція для вкладених функцій
    let numOne, numTwo;

    function getNumOne () { //вкладена функція (дочірня)
        numOne = 1;
    }

    function getNumTwo () { //вкладена функція (дочірня)
        numTwo = 2;
    }

    getNumOne ();
    getNumTwo ();

    let numSumm = numOne + numTwo;

    alert(numSumm);
}

getSumm ();

Спроба викликати дочірню функцію за межами батьківської виклике помилку.

У режимі "use strict", якщо функція оголошена всередині якогось блоку, то при спробі викликати її за межами цього блоку виникне помилка.

Локальні, зовнішні та глобальні змінні

Локальна змінна - це змінна, що оголошена всередині функції і видима лише всередині цієї функції. 

                
function showMessage () {
    let message =  "Hello!"; //локальна змінна
    
    alert (message);
}

showMessage ();
alert (message); // буде помилка

Зовнішня змінна - це змінна, що оголошена поза межами функції. Функція має повний доступ до зовнішніх змінних і може змінювати їхнє значення. 

                
let userName = "John"; //зовнішня змінна

function showMessage () {
    userName = "Jack"; // змінюємо значення зовнішньої змінної

    let message = "Hello, " + userName;
    alert (message);
}

alert (userName); // John - перед викликом функції
showMessage (); // Hello Jack
alert (userName); //Jack - функція змінила значення зовнішньої змінної

Зовнішня змінна використовується, якщо у функції немає такої локальної. Якщо ж у функціїї є однойменна локальна змінна, то вона перекриє зовнішню і зовнішня змінна буде проігнорована. 

                
let userName = "John"; //зовнішня змінна

function showMessage () {
    let userName = "Jack"; // оголошена однойменна локальна змінна

    let message = "Hello, " + userName; 
    alert (message);
}
// фукнція створить свою власну локальну змінну userName
showMessage (); // Hello, Jack
alert (userName); //John - зовнішня змінна не змінилась

Глобальні змінні - це змінні, що оголошені зовні усіх функцій. Глобальні змінні видимі для будь-якої функції, якщо тільки їх не перекривають однойменні локальні змінні. Бажано зводити використання глобальних змінних до мінімуму. В сучасному коді зазвичай є мало або ж взагалі немає глобальних змінних. Хоч вони іноді корисні для збереження найважливіших загальнопроектних даних. 

                
let globalVar = "Глобальна змінна";

function getSum () {
    let numOne, numTwo;

    function getNumOne () {
        numOne = 1;
        alert (globalVar); //Глобальна змінна
    }

    function getNumTwo () {
        numTwo = 2;
    }

    getNumOne ();
    getNumTwo ();

    let numSum = numOne + numTwo;

    alert (numSum);
    alert (globalVar); //Глобальна змінна
}

alert (globalVar); //Глобальна змінна
getSum ();
